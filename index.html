How I Built This 🚀

This entire website was built using Claude (by Anthropic) as my AI coding tool. No code was written manually — everything was generated through prompting. Below is a timeline of the key prompts I used and what each one produced.

Step 01 — Defining the Project
PROMPT "I need assistance in my Data Structures and Algorithms Final Project. I want the website to feature a mouse clicker game such as cookie clicker, and I want it uploaded to GitHub so it is available to the public. You must do everything on your own, such as coding and deployment."
Claude read the assignment PDF and confirmed all requirements: build using AI only, no manual coding, deploy publicly, include a process page. It asked clarifying questions about theme, users, and page style before writing any code — making sure the full scope was understood first.
Step 02 — Choosing Design & Features
PROMPT "🌙 Moon / Space theme. Multiple users should be able to play — not just me. Dark themed with a light mode toggle (sun or moon emoji) top right. Mobile friendly on ALL resolutions with tap gestures. Save progress per browser."
Claude locked in the full feature set: space-themed idle clicker with localStorage for per-browser save data, a 🌙/☀️ theme toggle, fully mobile-responsive layout using pointer events for native tap support, floating click effects, milestone toast notifications, a leaderboard, and 12 unlockable upgrades.
Step 03 — Building the Game
PROMPT "Build the full game now based on everything we discussed."
Claude produced the complete game page. Features include: an animated starfield background, a glowing 3D moon with crater details and tap animation, a stats bar tracking stardust / per tap / per second, 12 space-themed upgrades (Lunar Lens → Satellite Array → Black Hole Trap → Galaxy Merger → Big Bang Tap) with scaling costs, auto-production per second, floating +N tap effects, milestone notifications, a per-browser leaderboard, and a reset option.
Step 04 — Building This Page
PROMPT "For the How I Built This section, can we make it a separate tab on the same HTML rather than redirecting to another page? And just show the important prompts — clean timeline format."
Claude merged both pages into a single index.html file with an in-page tab switcher. The "How I Built This" tab uses the same design language — starfield, theme toggle, Orbitron font, timeline layout — so the whole experience feels cohesive. No redirect, no second file needed.
Step 05 — Deployment
PROMPT "Deploy this to GitHub Pages so it is publicly accessible."
The completed index.html was uploaded to a public GitHub repository and published via GitHub Pages. The site went live at a public URL with no server, no cost, and no backend. Anyone with the link can play the game and view this page instantly.
